# Node.js

Node.js — серверний JavaScript-рантайм і стандартна бекенд-платформа каналу. Рекомендація Віктора однозначна: тим, хто вчить бекенд, він радить Node.js, а не Bun — це стандартне середовище, з набагато більшою кількістю документації та відповідей на Stack Overflow, і LLM знають його значно краще ([[qa-and-plans-for-2024]]). Його стек вибору для нового продукту — Node-бекенд + JS-фронтенд + React Native для мобільного, щоб уся команда говорила однією мовою; окремо він хвалить, наскільки просто в Node даються асинхронна робота і WebSocket.

Стандартний модуль `crypto` в Node — це ще і його навчальна лабораторія: всі робочі демо з відео про асиметричне шифрування — підпис/перевірка та шифрування/дешифрування RSA, підписи на еліптичних кривих, AES-256-CBC — це звичайні Node-скрипти, викладені на GitHub, із практичним попередженням: `Math.random()` небезпечний, а криптографічна випадковість має братися з `crypto.getRandomValues` ([[asymmetric-encryption-digital-signatures]]).

## Де розглядається
- [[asymmetric-encryption-digital-signatures]] — вбудований модуль `crypto` у Node живить кожне демо (RSA, підписи ECC, AES-256-CBC); Math.random() — не джерело безпечної випадковості.
- [[qa-and-plans-for-2024]] — Node замість Bun для тих, хто вчиться; Node-бекенд + JS-фронтенд + React Native як його одномовний стек; ергономіка async/WebSocket.
