# Google I/O 2023 Watch Party — що цікавого показав Google для розробників (livestream)

> Оригінал: "Google I⧸O 2023 Watch Party - що цікавого показав Google для розробників" — https://youtu.be/Zq-Pv4g3JC4

Спільний livestream із гостем **Микитою Галкіним** — інженером Google Cloud (команда Cloud Run) і організатором GDG — з оглядом анонсів Google I/O 2023, до якого автор упродовж усього ефіру додає коментарі як колишній Googler. Вони проходяться по залізу (Pixel 7a, Pixel Fold, Pixel Tablet — автор, самопроголошений фанат Apple, зізнається, що дизайн із камерною смугою «зачіпає його душу», і жартує про скандал із нейронним «покращенням» фото Місяця, де мережа вклеює запам’ятований Місяць поверх розмитого), а тоді заглиблюються в AI-історію: [[gemini|Bard]] переходить від урізаного LaMDA (обмежений запуск якого провалився) до PaLM 2, злиття Google Brain + DeepMind і стратегія вбудовувати LLM у Gmail/Docs/Cloud, а не постачати окремий чат. Автор ділиться власним бенчмарком: задача на Node.js — читати файл слово за словом через бінарні операції Buffer — ChatGPT-4 упорався ідеально, включно з тонким виправленням багу з async/await, тоді як Bard борсався «як джуніор, що написав би ChatGPT по допомогу». Далі Микита влаштовує тур по Google Cloud/Firebase: підтримка Terraform для конфігу Firebase, режими Firestore, Cloud Functions на контейнерах Cloud Run, rewrites у Firebase Hosting, що маршрутизують прямо на сервіси Cloud Run, маркетплейс Firebase Extensions, Cloud Workstations (браузерні dev-VM, де вихідний код ніколи не торкається вашого ноутбука — zero-trust замість VPN) і безпека ланцюга постачання: підписані образи, **Assured Open Source Software** від Google (перевірені Google, перезібрані й підписані OSS-пакети) і галочка provenance у npm. Родзинки веб-платформи: Chrome замінює замочок HTTPS на нейтральну іконку «tune», WebGPU і Core Web Vitals.

## Головне
- **Замочок — це брехня, яку ми нарешті прибираємо** ([[https-tls|HTTPS]]): Chrome міняє замочок на іконку налаштувань, бо [[security-practices|HTTPS]] лише каже, що канал зашифрований і сертифікат відповідає URL — «він не каже нічого про надійність контенту»; фішингові сайти отримують той самий замочок, а користувачі читають його як значок довіри.
- **Безпека ланцюга постачання отримала реальний інструментарій** ([[security-practices]]): поверхня атаки тягнеться від вашого репозиторію через білд-системи до деплою («хакер може сидіти у ваших білд-машинах»); серед показаних заходів — відтворювані білди (reproducible builds), підпис образів, який перевіряється під час деплою в Cloud Run, список перевірених пакетів Assured OSS від Google і npm provenance — хоча Микита зауважує, що підпис лише доводить, що *чийсь* ключ підписав, і JS-екосистему особливо важко валідувати.
- **Bard проти ChatGPT-4, вердикт практика** ([[ai-coding-agents]]): на його реальній задачі з бінарним файлом/Buffer із перемішаним синхронним/асинхронним кодом GPT-4 розв’язав те, що GPT-3.5 зробив неправильно, тоді як Bard не встигав. Йому все одно подобається чесність Bard — коли спитали, той зізнався, до яких даних мав доступ. Open-source LLM плодяться так швидко (згадується злитий меморандум «we have no moat»), що донавчання малих задачно-специфічних моделей виглядає майбутнім; ставка Google — інтеграція в продукти, а не окремий чат-бот.
- **Firebase + Cloud Run — рекомендований дешевий стек його співведучого:** Cloud Run запускає будь-який контейнер зі scale-to-zero, метриками і CI/CD «у два кліки» (платформа-портфоліо доповідачів mytalks.tech від Микити працює на ньому ~у 3 рази дешевше за DigitalOcean); Firebase Hosting бере на себе домени/сертифікати і маршрутизує `/api`-роути на сервіси Cloud Run; конфіги Firebase тепер можна імпортувати в Terraform, тож infra-as-code нарешті їх покриває. Автор лишається людиною реляційних БД — після болючих міграцій MongoDB він «просто бере SQL-базу» для нових проєктів, визнаючи водночас, що адмінка Firestore і realtime-синхронізація чудові для прототипів.
- **[[webgpu|WebGPU]] — тиха вбивча фіча:** WebGL був урізаною гілкою 25-річного OpenGL, і запуск нейромереж у браузері означав знущання над шейдерами — кодування даних у кольори пікселів, обчислення «на милицях» і молитви, щоб anti-aliasing не зіпсував піксель. WebGPU дає прямий compute-доступ до GPU (Game of Life влазить в один Colab-подібний файл) із ~3× продуктивності для in-browser ML на кшталт видалення фону через TensorFlow.js — «ще один спосіб майнити біткоїн у браузерах ваших відвідувачів», жартує він.
- **[[web-performance|Core Web Vitals]], огляд із запереченнями:** Google зійшовся на трьох користувацько-орієнтованих метриках — LCP (основний контент ≤2.5 с), CLS (без стрибків верстки) і латентність взаємодії (~100 мс; INP замінює FID) — плюс підказки `fetchpriority`, бо старі метрики PageSpeed накручували, а користувачі все одно дивилися, як нестилізований контент смикається на місце. Далі пара наживо аудитує сайти (сайт спільноти Odesa JS має гарний бал); контраргумент автора: для залогінених «товстих клієнтів» за авторизацією думка Google важить менше за думку ваших користувачів — гонитва за метриками заради SEO — це для контентних сайтів.
- **Колорит колишнього Googler’а** про те, чому працює монорепозиторій: одна версія всього («найсвіжіший Angular для *всіх* проєктів»), тож будь-хто, хто оновлює залежність, мусить лишити *ваші* тести зеленими — а це ще й чому все має мати тести. Плюс історія походження [[livr|LIVR]]: WebbyLab стандартизує стеки між проєктами, спробували кожну бібліотеку валідації, всі виявилися недостатніми, написали спочатку специфікацію — і тепер той самий формат правил має реалізації на JS, PHP, Perl, Go, Swift і Erlang.
