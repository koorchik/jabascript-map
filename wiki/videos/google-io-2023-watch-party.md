# Google I/O 2023 Watch Party — What Google Showed for Developers (livestream)

> Original: "Google I⧸O 2023 Watch Party - що цікавого показав Google для розробників" — https://youtu.be/Zq-Pv4g3JC4

Joint livestream with guest **Mykyta Galkin** — a Google Cloud engineer (Cloud Run team) and GDG organizer — reviewing Google I/O 2023 announcements, with the author adding ex-Googler commentary throughout. They cover the hardware (Pixel 7a, Pixel Fold, Pixel Tablet — the author, a self-confessed Apple fan, admits the camera-bar design "touches his soul", and jokes about the Moon-photo neural "enhancement" scandal where the network pastes a remembered Moon over a blurry one), then dig into the AI story: [[gemini|Bard]] moving from a cut-down LaMDA (whose limited launch flopped) to PaLM 2, the Google Brain + DeepMind merger, and the strategy of embedding LLMs into Gmail/Docs/Cloud rather than shipping a chat box. The author shares his own benchmark: a Node.js task reading a file word-by-word via binary Buffer operations — ChatGPT-4 nailed it including a subtle async/await bug fix, while Bard flailed "like a junior who'd ping ChatGPT for help". Mykyta then tours Google Cloud/Firebase: Terraform support for Firebase config, Firestore modes, Cloud Functions running on Cloud Run containers, Firebase Hosting rewrites routing straight to Cloud Run services, Firebase Extensions marketplace, Cloud Workstations (browser-dev VMs where source never touches your laptop — zero-trust instead of VPN), and supply-chain security: signed images, Google's **Assured Open Source Software** (Google-vetted, rebuilt and signed OSS packages), and npm's provenance checkmark. Web platform highlights: Chrome replacing the HTTPS padlock with a neutral "tune" icon, WebGPU, and Core Web Vitals.

## Key takeaways
- **The padlock is a lie we finally retire** ([[https-tls]]): Chrome swaps the padlock for a settings icon because [[security-practices|HTTPS]] only says the channel is encrypted and the cert matches the URL — "it says nothing about the trustworthiness of the content"; phishing sites get the same padlock, and users read it as a trust badge.
- **Supply-chain security got real tooling** ([[security-practices]]): the attack surface runs from your repo through build systems to deploy ("a hacker can sit in your build machines"); mitigations shown include reproducible builds, image signing verified at Cloud Run deploy time, Google's Assured OSS list of vetted packages, and npm provenance — though Mykyta notes a signature only proves *someone's* key signed it, and the JS ecosystem is especially hard to validate.
- **Bard vs ChatGPT-4, practitioner's verdict** ([[ai-coding-agents]]): on his real binary-file/Buffer task with mixed sync/async code, GPT-4 solved what GPT-3.5 got wrong, while Bard couldn't keep up. He still likes Bard's honesty — when asked, it admitted what data it had access to. Open-source LLMs are proliferating so fast (the leaked "we have no moat" memo comes up) that fine-tuning small task-specific models looks like the future; Google's play is integration into products, not a standalone chatbot.
- **Firebase + Cloud Run is his co-host's recommended cheap stack:** Cloud Run runs any container with scale-to-zero, metrics and CI/CD "in two clicks" (Mykyta's speaker-portfolio platform mytalks.tech runs on it ~3× cheaper than DigitalOcean); Firebase Hosting handles domains/certs and rewrites `/api` routes to Cloud Run services; Firebase configs are now Terraform-importable so infra-as-code finally covers it. The author remains a relational-DB person — after painful MongoDB migrations he "just takes a SQL database" for new projects, while conceding Firestore's admin UI and realtime sync are great for prototypes.
- **[[webgpu|WebGPU]] is the quiet killer feature:** WebGL was a cut-down 25-year-old OpenGL lineage, and running neural nets in the browser meant abusing shaders — encoding data into pixel colors, computing "with crutches", and praying anti-aliasing didn't corrupt a pixel. WebGPU gives direct compute access to the GPU (a Game-of-Life fits in a Colab-style one-pager) with ~3× performance for in-browser ML like TensorFlow.js background removal — "one more way to mine Bitcoin in your visitors' browsers", he jokes.
- **[[web-performance|Core Web Vitals]] recap with pushback:** Google converged on three user-centric metrics — LCP (main content ≤2.5 s), CLS (no layout jumping), and interaction latency (~100 ms; INP replacing FID) — plus `fetchpriority` hints, because old PageSpeed metrics got gamed while users still watched unstyled content twitch into place. The pair then live-audit sites (Odesa JS community site scores well); the author's contrarian take: for logged-in "thick client" apps behind auth, Google's opinion matters less than your users' — SEO-driven metric-chasing is for content sites.
- **Ex-Googler color** on why the monorepo works: one version of everything ("the freshest Angular for *all* projects"), so anyone updating a dependency must keep *your* tests green — which is also why everything must have tests. Plus the origin story of [[livr|LIVR]]: WebbyLab standardizes stacks across projects, tried every validation library, found all wanting, wrote a spec first — and now the same rule format has implementations in JS, PHP, Perl, Go, Swift and Erlang.
